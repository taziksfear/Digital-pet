# Цифровой питомец "Envelope" (мы выиграли кста)

Проект команды **envelope()** для хакатона Digital Pet.  
Это интерактивный веб-питомец с 3D-моделями, голосовым управлением на базе AI, влиянием реальной погоды, мини-играми и системой прокачки.

## Идея

Мы создали виртуального питомца, который живёт в браузере и обладает собственным характером.  
Питомец:

- имеет имя и три базовые характеристики: сытость, энергия, настроение (плюс шкала туалета);
- реагирует на действия пользователя: покормить, поиграть, уложить спать, отвести в туалет;
- меняет своё состояние со временем — если за ним не ухаживать, он грустит, устаёт и может заболеть;
- отображается в 3D с разными анимациями (idle, сон, радость, болезнь и т.д.);
- умеет разговаривать — пользователь может нажать на микрофон и задать вопрос, а питомец ответит уникальной фразой, сгенерированной AI (через Ollama);
- реагирует на реальную погоду в городе пользователя (определяется автоматически) — в дождь настроение падает быстрее, а питомец может грустно прокомментировать;
- имеет гардероб с разными персонажами (скинами), которых можно открывать в колесе фортуны;
- содержит мини-игры (слоты) для заработка внутриигровых монет;
- ведёт учёт заданий (квестов), за выполнение которых даются награды.

## Технологии

- **Frontend**: React + Vite, Three.js (react-three-fiber), CSS-in-JS
- **Backend (игровая логика)**: Go (1.22+), SQLite
- **AI / Voice**: Python (FastAPI), Vosk (распознавание речи), Ollama (генерация текста)
- **Внешние API**: Open-Meteo (погода, геокодинг)

## Инструкция по запуску

### Требования

- Node.js 18+
- Go 1.22+
- Python 3.10+
- Установленная модель Vosk (small-ru) и Ollama с моделью `gpt-oss:120b-cloud`

### 1. Клонирование репозитория

```bash
git clone https://github.com/taziksfear/Digital-pet.git
cd Digital-pet
```

### 2. Запуск frontend (React)

```bash
cd frontend
npm install
cp .env.example .env   # отредактируйте при необходимости
npm run dev
```

Фронтенд будет доступен на `http://localhost:5173`.

### 3. Запуск backend на Go

```bash
cd backend-go
go mod tidy
go run main.go
```

Сервер запустится на `http://localhost:8080`.

### 4. Запуск AI-сервера на Python

```bash
cd backend-python
pip install -r requirements.txt
# Скачайте модель Vosk и поместите в папку vosk-model-small-ru-0.22
python main.py
```

Сервер будет слушать порт `8000` (`http://localhost:8000`).

### 5. Переменные окружения

Создайте файл `.env` в папке `frontend`:

```env
VITE_GO_API_URL=http://localhost:8080
VITE_PYTHON_API_URL=http://localhost:8000
VITE_DEV_PASSWORD=1235   # пароль для входа в dev-режим
```

Для Go и Python переменные не требуются (все настройки внутри кода).

## Особенности реализации

- **3D-анимации** — каждая модель персонажа содержит набор анимаций (idle, sleep, joy_jump, hello, fall, sick, eat, toilet), которые переключаются в зависимости от состояния питомца и действий пользователя.
- **Голосовое общение** — запись голоса через браузер, отправка на Python-сервер, распознавание Vosk, генерация ответа через Ollama, возврат текста и отображение в «облачке».
- **Погода** — при запуске приложение определяет геолокацию пользователя (или использует IP), получает реальную погоду через Open-Meteo и отправляет её на Go-сервер. Сервер учитывает погоду при расчёте динамики настроения (в дождь падает быстрее).
- **Гардероб и рулетка** — персонажи хранятся в JSON-списке, разблокированные сохраняются в БД. Колесо фортуны реализовано как бесконечная лента с призами (персонажи, монеты).
- **Синхронизация состояния** — клиент каждые 3 секунды опрашивает сервер и получает актуальные значения характеристик; все изменения stat-ов происходят на сервере (воркер с тикером 1 сек).
